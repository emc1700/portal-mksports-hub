# portal-mksports-hub

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文档的形式存放，不针对任何特定域名或具体网站。项目的主要目的是提供一个集中存放和展示各类 HTML 页面的平台，便于访问和浏览。

## 目录说明

- `pages/`：存放所有独立的 HTML 页面文件。每个页面为一个独立的静态文档，可直接在浏览器中打开。
- `assets/`：存放页面所需的静态资源，如图片、样式表、脚本等。资源文件按页面或功能分类存放。
- `index.html`：仓库首页，提供对 pages 目录下各页面的索引或导航。

## 页面归档说明

1. **归档范围**：本仓库收录的 HTML 页面均为独立、自包含的静态文档。页面内容不涉及特定网站或域名，仅作为通用展示或功能演示。
2. **文件命名**：页面文件名建议使用英文小写，以连字符分隔单词，例如 `example-page.html`。
3. **资源引用**：页面中引用的资源（CSS、JavaScript、图片等）应尽量使用相对路径，指向 `assets/` 目录下的对应文件，以保证页面在本地或服务器上均可正常显示。
4. **兼容性**：页面应尽量遵循现代 Web 标准，确保在主流浏览器中能够正常渲染。

## 维护说明

- **添加新页面**：将新的 HTML 文件放入 `pages/` 目录，并确保其引用的资源文件已放置在 `assets/` 对应子目录中。如有需要，可更新 `index.html` 中的索引链接。
- **更新现有页面**：直接修改对应文件即可。建议在提交时附上简要的变更说明。
- **删除页面**：从 `pages/` 目录中移除文件，并同步清理 `assets/` 中不再使用的资源文件，同时更新 `index.html` 中的链接。
- **提交规范**：提交信息应简洁明了，描述本次变更的内容或原因。
